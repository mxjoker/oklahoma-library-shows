## On-Page SEO Rules (Apply to Every Page)

### Title Tag Formula
```
[Primary Keyword] | Oklahoma Library Shows — Funky Monkey Events
```
Examples:
- `Magic Show for Libraries in Oklahoma | Oklahoma Library Shows`
- `2027 Summer Reading Program Entertainer | Oklahoma Library Shows`
- `Library Shows in Tulsa, OK | Oklahoma Library Shows`

Max 60 characters before the pipe. Always include "Oklahoma" or city name.

### Meta Description Formula
```
[Hook with primary keyword]. [Value prop]. [CTA with phone number].
```
Max 155 characters. Include: keyword, Joe's name or Funky Monkey, and a CTA.
Example: `Book a magic show for your Oklahoma library's 2027 summer reading program.
Joe Coover brings 6 interactive programs. No travel fee in OKC + Tulsa. Call (405) 431-6625.`

### H1 Rule
Every page gets exactly one H1. It must contain the primary keyword naturally.
No generic H1s like "Welcome" or "About Us."

### Internal Linking Rules
- Every show page links to: booking.html, 2027-summer-reading.html, and 2 other show pages
- Every location page links to: all 6 show pages + booking.html
- Homepage links to: all show pages, all major location pages, 2027 page, resources
- 2027 page links to: all show pages + booking.html

### Image SEO Rules
- Every image must have a descriptive alt tag: `alt="Joe Coover performing magic show at [Library Name] in Oklahoma City"`
- File names should be descriptive: `magic-show-tulsa-library.jpg` not `IMG_4392.jpg`
- Use WebP format where possible for performance
- See docs/photo-brief.md for specific photo requests per page

## Competitive Moat Strategy
1. **Publish first on 2027 theme** — no competitor will have content targeting "Mysteries Await" yet
2. **Transparent pricing** — librarians respect $385 upfront; competitors hide it
3. **Geographic specificity** — city and rural cluster pages that no competitor has
4. **Librarian language** — use CSLP, ODL, ALA terminology throughout; signals expertise
5. **Resource hub** — downloadable guides build backlinks when librarians share them
6. **Email list** — newsletter capture on resources means repeat booking without re-ranking
